Le principe

Comment ça marche

  1. Structure : ajoutez vos sections et vos champs (16 types, du texte simple à la signature manuscrite)
  2. Apparence : choisissez vos couleurs (10 palettes dégradés webgradients) et votre logo
  3. Livraison : définissez l'URL courte, vos destinataires mail et la durée de conservation (30 à 365 jours)
  4. Publication : vérifiez le récap et publiez à keystone.app/f/{votre-slug}
  5. Réponses : consultez les réponses reçues depuis la bibliothèque (bouton œil), exportez en CSV
Bon à savoir

Questions fréquentes

Combien de temps les réponses sont-elles conservées ?
Selon la TTL que vous avez choisie (30, 90, 180 ou 365 jours). Après expiration, suppression automatique côté serveur — conformité RGPD Art. 5 native. Le mail Resend reste dans la boîte du destinataire indéfiniment.
Mes répondants doivent-ils créer un compte ?
Non, le formulaire est ouvert et anonyme par défaut. Vous pouvez activer un code d'accès optionnel dans l'étape Apparence pour les questionnaires confidentiels.
Comment partager le formulaire avec mes répondants ?
Une fois publié, copiez l'URL keystone.app/f/{votre-slug} et envoyez-la par mail, SMS, Slack ou QR code. Aucune installation nécessaire pour le répondant.
Puis-je dupliquer un formulaire existant ?
Oui : dans la bibliothèque, bouton 'Dupliquer'. Idéal pour réutiliser une structure (par exemple Biennale 2026 → Biennale 2027) en ne changeant que les destinataires et quelques champs.
Comment fonctionne la logique conditionnelle ?
Sélectionnez un champ dans le builder, puis dans l'éditeur de droite ouvrez 'Visibilité conditionnelle' ou 'Obligatoire conditionnel'. Vous pouvez aussi configurer une somme automatique entre champs Montant.
Puis-je pré-remplir des champs via l'URL ?
Oui : chaque champ a un ID technique (visible dans son éditeur) qui peut être utilisé en query param. Exemple : keystone.app/f/biennale?fld_nom=Marie+Durand.
Quels types de fichiers les répondants peuvent-ils joindre ?
Pas d'upload direct. À la place, utilisez le type 'Lien externe' : le répondant colle un lien WeTransfer, Drive, Dropbox ou Vimeo.
Le guide

Créer un formulaire sans coder (et conforme RGPD)

Recueillir une inscription, une demande de devis ou un avis ne devrait pas exiger un développeur. Mais les solutions grand public posent une question qu'on oublie souvent : où partent les données que vos répondants vous confient ?

L'alternative aux formulaires classiques

Construire un formulaire est aujourd'hui à la portée de tous : on choisit ses champs, on rédige ses questions, on partage un lien. Pas de code, pas d'hébergement à gérer. Le vrai sujet n'est plus technique, il est juridique.

Pourquoi le RGPD compte ici

Un formulaire collecte presque toujours des données personnelles — nom, e-mail, parfois plus. Beaucoup d'outils gratuits stockent ces réponses sur des serveurs hors d'Europe, les analysent, voire les monétisent. Vous restez pourtant responsable de ce que deviennent les données de vos répondants.

  • Consentement clair : la personne sait à quoi elle s'engage en répondant.
  • Données maîtrisées : pas de revente, pas de traceurs publicitaires greffés sur votre formulaire.
  • Hébergement maîtrisé : savoir où les réponses sont conservées, et pouvoir les supprimer.
  • Minimisation : ne demander que les champs réellement utiles. Moins vous collectez, moins vous portez de risque.

Ces principes ne sont pas un luxe juridique : ils protègent vos répondants et vous évitent d'engager votre responsabilité sans le savoir, pour un simple formulaire de contact.

Partager simplement

Un bon formulaire se diffuse par un simple lien ou un QR code — sur une affiche, un e-mail, un comptoir. Le répondant ouvre, remplit, valide ; vous récupérez des réponses propres, sans friction.

Key Form réunit ces deux exigences : la simplicité d'un formulaire sans code et la conformité RGPD par défaut, partageable par lien ou QR.
