Pipeline Paiement — Stripe Webhook
💳
Flux complet (automatique)
💳
Client paie sur Stripe
Via Payment Link (Starter / Pro / Max)
⚡
Webhook → Worker
POST /api/stripe/webhook · Signature HMAC vérifiée · Idempotence via stripe_events
🔑
Génération clé licence
Format XXXX-XXXX-XXXX-XXXX · CSPRNG · Hashée PBKDF2 100k iterations + blind index HMAC
🗄️
Insertion D1
Clé + hash + plan + stripe_customer_id + stripe_subscription_id + email
📧
Email Resend
De: noreply@protein-keystone.com · BCC: protein.keystone@gmail.com · Lien activation direct
✅
Activation client
Clic lien → protein-keystone.com/?ks_key=XXXX → auto-remplissage → JWT 30j → vault sync
Events Stripe traités
checkout.session.completed Génère + envoie clé
customer.subscription.deleted Désactive licence (is_active=0)
customer.subscription.updated Upgrade/downgrade plan
Sécurité API
Authentification JWT HS256 · TTL 30 jours · stocké localStorage
Hachage clés PBKDF2-SHA256 · 100 000 iterations + sel aléatoire
Recherche HMAC blind index (lookup_pepper) — pas de clé en clair en DB
Vault chiffré AES-256-GCM · IV aléatoire · max 64 KB
CORS Multi-origin whitelist · preflight OPTIONS
Rate-limit Tentatives activation loggées par IP

Format des clés de licence
🔑
Format standard
XXXX-XXXX-XXXX-XXXX
• 16 caractères + 3 tirets
• Alphabet 32 chars (sans I, O, 0, 1, L pour éviter confusions)
• Généré via CSPRNG (crypto.getRandomValues)
• Stocké uniquement en forme hachée en base
Commandes de gestion
# Redéployer le Worker cd workers && npx wrangler deploy # Mettre à jour un secret npx wrangler secret put NOM_SECRET # Appliquer migration DB npx wrangler d1 execute keystone-os \ --file=./src/db/migration_sprint5.sql \ --remote # Voir les logs Worker npx wrangler tail keystone-os-api # Push vers production git push origin main
